# Oh-My-Life-Patchy 架构说明

## 1. 总体结构

Oh-My-Life-Patchy 采用：

**1 个主 Agent + 8 个 Skills**

主 Agent 负责：
- 识别当前问题类型
- 判断属于哪类日常隐性损耗
- 路由到对应 skill
- 在复杂场景中进行多技能协同编排

---

## 2. 主 Agent 的核心能力

### 2.1 损耗诊断
将用户问题识别为以下类型之一：
- 沟通损耗
- 行动损耗
- 判断损耗
- 生活摩擦

### 2.2 技能路由
将问题自动分发到最合适的 skill。

### 2.3 复合场景编排
当一个问题同时涉及：
- 沟通
- 信息理解
- 启动困难
- 截止止损
时，主 Agent 会按优先级进行组合处理。

---

## 3. 8 个 Skills

### 沟通补丁
- 回消息急救
- 体面表达
- 退出机制

### 行动补丁
- 起手代打
- 未来减麻烦

### 判断补丁
- 行动翻译器
- 后悔模拟器

### 生活补丁
- 找东西推理

---

## 4. 当前重点实现

当前版本重点实现：
- 回消息急救
- 起手代打
- 行动翻译器

这三个 skill 共同支撑最核心的高频问题：
- 消息拖延
- 启动困难
- 通知理解困难

---

## 5. 复合场景处理顺序

在高难复合场景中，主 Agent 默认按以下顺序处理：

1. 识别问题结构
2. 先生成状态同步消息
3. 再翻译任务要求
4. 再给时间受限下的止损计划
5. 最后指出最容易翻车的点

---

## 6. 扩展方向

未来版本可进一步扩展为具身化生活补丁终端，接入：
- 麦克风与低成本语音转写
- 语音播报模块
- 便携玩偶或微型终端
- 微型打印模块
- 移动载体
